> 本文档专注于可视化分析工作台的 UI 布局、交互设计与核心功能思路。

## 二、顶部菜单栏

```
[← 返回]  [项目名称]                    [数据选择] [导出] [分享] [保存]
──────────────────────────────────────────────────────────────────
[+ 新建关键帧] [工具: 选择▾] [测量▾] [标注▾]    [2D/3D] [跟随自车] [重置视角]
```

**第一行（全局操作）：**
- 返回数据管理平台
- 项目名称（可编辑）
- 数据选择：打开 DataSelection 配置（选择哪些录包、对象、时间段）
- 导出：导出当前帧截图、导出分析报告
- 分享：生成可分享链接（带当前时刻和视角）
- 保存：保存当前分析会话

**第二行（画布工具条）：**
- 新建关键帧：在当前时刻插入问题关键帧
- 工具下拉：选择模式、测量距离、框选区域
- 标注下拉：添加文字标注、箭头标注
- 视图控制：2D/3D 切换、跟随自车模式、重置视角

## 四、中间画布区域

### 4.1 视图 Tab

```
[🗺 2D 鸟瞰]  [🧊 3D 场景]  [🧭 导航地图]  [📷 相机视角]  [+ 添加视图]
```

每个 Tab 是独立的 Viewer 实例，共享同一个 Player 的时间状态。

**2D 鸟瞰（Canvas 2D）**
- 俯视视角，BEV（Bird's Eye View）
- 支持平移、缩放、旋转
- 渲染：车道线、障碍物框、自车轨迹、点云投影
- 可选：跟随自车自动平移

**3D 场景（Three.js）**
- 三维视角，可自由旋转
- 渲染：3D 障碍物框、点云、预制体模型
- 支持透视/正交投影切换

**导航地图（高德/Cesium）**
- 叠加在真实地图上
- 显示 GPS 轨迹、地理位置
- 适合分析路网级别的问题

**相机视角（Image/Video）**
- 显示某个相机的图像/视频流
- 可叠加检测框（来自 obstacle 对象的 2D 投影）
- 多相机时可分屏

### 4.2 画布内悬浮控件

```
画布右上角：
  [⊕ 放大] [⊖ 缩小] [⌂ 重置] [□ 全屏]

画布左上角（当前视图信息）：
  t = 45.23s    frame #452    速度 1x

画布右下角（坐标显示）：
  x: 12.45m  y: 3.21m  heading: 0.52rad
  （鼠标悬停时实时更新）
```

### 4.3 对象悬停与点选

```
鼠标悬停对象：
  └── 显示 Tooltip：ID / 类型 / 关键属性值

点击对象：
  └── 高亮该对象（边框加粗）
  └── 右侧属性面板切换到该对象
  └── 左侧对象列表滚动到该行

右键对象：
  ├── 查看属性
  ├── 在时间轴中定位
  ├── 添加标注到此对象
  ├── 创建 Issue（关联此对象 + 当前时刻）
  └── 导出此对象时间序列数据
```

## 六、问题关键帧系统

### 6.1 什么是问题关键帧

分析人员在播放过程中，发现某一时刻有问题，**插入一个标记帧**，记录问题描述和问答。

- 区别于普通书签：有结构化的问题描述、严重程度、问答字段
- 可批量转换为 Issue
- 在时间轴上有专属样式（红/黄标记）
- Player 可在问题帧之间快速跳转

### 6.2 创建问题关键帧

三种方式：
1. 顶部工具条 → [+ 新建关键帧]
2. 播放暂停时 → 快捷键 `K`
3. 右键画布/对象 → "标记此帧为问题"

创建后弹出快速填写卡：

```
⚠️ 问题关键帧  t = 45.23s
─────────────────────────────
标题    [___________________________]
严重度  ○ info  ● warning  ○ error  ○ critical
关联    obstacle:42  [×]  [+ 添加对象]

[稍后填写]            [完成]
```

### 6.3 右侧属性面板：问题关键帧详情

```
⚠️ 问题关键帧
─────────────────────────────
时间         45.23s
严重程度     [error ▼]
标题         [车道线置信度骤降_________]

关联对象
  lane_line:left_01  [×]
  ego_vehicle        [×]
  [+ 添加对象]

问题描述
  [在弯道处，左侧车道线置信度从0.92
   骤降至0.41，持续约2秒__________
   _________________________________]

问答（结构化分析）
  Q: 哪个模块的问题？
  A: [感知模块 ▼]

  Q: 初步判断原因？
  A: [光照/遮挡 ▼]

  Q: 是否可复现？
  A: [是，多次出现 ▼]

  Q: 优先级？
  A: [P1 - 高优先 ▼]

截图
  [🖼 自动截图预览]
  [重新截图]

─────────────────────────────
[→ 转为 Issue]    [删除]
```

**问答题目可在平台配置中自定义**，不同项目可配置不同的分析问卷。

### 6.4 批量管理问题关键帧

左侧对象面板增加专属分组：

```

  ⚠️ t=12.3s  车道线丢失        [error]
  ⚠️ t=23.8s  障碍物漏检        [critical]
  ⚠️ t=45.2s  置信度骤降        [warning]
  ⚠️ t=67.1s  定位漂移          [error]
  ...
  [全部转为 Issue]   [导出报告]
```

## 七、播放器与时间轴

### 7.1 播放器控制栏

```
┌──────────────────────────────────────────────────────────────────┐
│  [⏮] [◀◀] [◀] [▶/⏸] [▶] [▶▶] [⏭]     0.5x [1x▾] 2x 4x      │
│                                                                    │
│  [⚠◀] [⚠▶]  问题帧 3/8              [⏱ 00:45.23 / 05:00.00]  │
└──────────────────────────────────────────────────────────────────┘
```

**控制按钮说明：**

```
[⏮]   跳到开头
[◀◀]  快退 10s
[◀]   上一帧
[▶/⏸] 播放 / 暂停
[▶]   下一帧
[▶▶]  快进 10s
[⏭]   跳到结尾

[⚠◀]  跳到上一个问题关键帧
[⚠▶]  跳到下一个问题关键帧
  3/8   当前是第 3 个，共 8 个问题帧
```

**跳转到问题帧时：**
- Player 自动暂停
- 右侧属性面板切换到该关键帧的详情
- 画布聚焦到关联对象（如有）
- 时间轴该帧高亮

### 7.2 时间轴

```
时间轴（可缩放，可拖拽）

0s                    30s                   60s              120s
│────────────────────────────────────────────────────────────────│
│  进度条（可拖拽 seek）                                          │
│                                                                │
│  ego         ████████████████████████████████████████████    │ ← 对象存在范围
│  obstacle:42      ████████████████████████████████           │
│  obstacle:43            ██████████████████                   │
│  lane:left   ████████████████████████████████████████████    │
│                                                                │
│  ⚠           ·         ⚠              ⚠           ⚠          │ ← 问题关键帧
│              12s        38s            67s         98s         │
└────────────────────────────────────────────────────────────────┘
```

**时间轴交互：**
- 拖拽进度条 → seek 到指定时刻
- 点击对象行 → 左侧面板/右侧面板显示该对象
- 点击 ⚠ 标记 → 跳转到该问题帧并显示详情
- 滚轮 → 时间轴缩放（放大看细节/缩小看全局）
- 对象行右键 → 固定到时间轴 / 取消固定
- 拖拽对象行高度 → 调整该行高度（点云/图像行需要更高）

**固定到时间轴：**
不是所有对象都显示在时间轴，默认只显示：
- 自车
- 分析人员手动固定的对象
- 有问题关键帧的对象

### 7.3 时间轴上的图层

```
时间轴图层（从上到下）

  ─ 进度条 + 时间刻度
  ─ 问题关键帧层（所有 ⚠ 标记，始终显示）
  ─ 固定对象层（用户手动固定的对象轨迹）
  ─ 选中对象层（当前选中对象，自动显示）
```

## 八、分析会话记录

每次打开录包进行分析，自动创建一个分析会话，记录所有操作：

```typescript
// 自动记录的事件（用户无感知）
type AnalysisEvent =
  | { type: 'OPEN_RECORDING';    recording_id: string }
  | { type: 'SEEK_TO';           time: number }
  | { type: 'PAUSE_AT';          time: number; dwell_ms: number }
  | { type: 'SELECT_OBJECT';     object_id: string; time: number }
  | { type: 'ZOOM_IN';           area: BBox; time: number }
  | { type: 'ADD_KEYFRAME';      keyframe_id: string; time: number }
  | { type: 'CREATE_ISSUE';      issue_id: string; from_keyframe: string }
  | { type: 'SWITCH_VIEW';       from: string; to: string }
  | { type: 'COMPARE_OPEN';      recording_b: string }
```

会话结束后（关闭页面或手动结束），会话数据上传到服务器，用于：
- 追溯分析过程
- AI 学习人类分析模式
- 生成"分析报告"时还原分析步骤

## 十二、状态说明

```
平台整体状态机

IDLE                 没有加载录包
    ↓ 打开录包 + 选择 DataSelection
PREPARING            后端正在处理数据
    ↓ 处理完成
READY                数据就绪，前端开始加载对象
    ↓ 对象加载完成
ANALYZING            正常分析状态
    ├── 播放中（Player running）
    ├── 暂停中（Player paused）
    └── 跳帧（Player seeking）

ANALYZING 下的子状态：
  HAS_SELECTION      有选中对象
  COMPARE_MODE       对比模式（加载了第二个录包）
  KEYFRAME_DETAIL    正在查看/编辑问题关键帧
```

## 十三、与其他模块的集成点

```
← 数据管理平台
  入口：点击录包 → 打开分析工作台
  传入：recording_id, 可选的 time_range

→ Issue 管理（Plane）
  出口：问题关键帧 → 一键创建 Issue
  传入：recording_id + time + object_ids + snapshot + 问答内容

→ 项目管理平台
  出口：Issue 自动归属到对应项目里程碑
```
