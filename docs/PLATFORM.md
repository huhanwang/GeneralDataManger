# 平台架构总览

> 三大平台统一工具链，数据驱动开发全生命周期管理。

---

## 一、平台全景

```
┌─────────────────────────────────────────────────────────────────────┐
│                        统一入口 / 工作台                              │
├──────────────────┬──────────────────────┬───────────────────────────┤
│   数据管理平台    │    项目管理平台        │     人员管理平台           │
│  Data Platform   │  Project Platform    │   Personnel Platform      │
│                  │                      │                           │
│  核心：数据资产   │  核心：问题与进度      │   核心：工作与效能         │
│  的管理与分析     │  的跟踪与流转          │   的记录与汇总             │
└──────────────────┴──────────────────────┴───────────────────────────┘
                          ↕ 共享数据模型
              Recording / Object / Issue / Session / User
```

三个平台**共用一套底层数据**，不同平台是对同一份数据的不同视角和聚合方式。

## 二、数据管理平台

### 2.1 定位

管理所有数据资产，从上传到分析到归档的完整生命周期。

### 2.2 功能模块

```
数据管理平台
│
├── 📁 数据库（Data Library）           ← 核心入口
│   管理所有录包/数据集
│   上传、检索、标签、版本、归档
│
├── 🔬 可视化分析工作台                 ← 核心子功能（见 VIZ_WORKBENCH.md）
│   打开录包进行深度分析
│   对象可视化、关键帧标记、问题发现
│
├── 📊 统计报告（Report）
│   单录包分析报告
│   多录包对比报告
│   算法版本对比（before/after）
│   问题趋势统计
│
└── ⚙️ 数据配置（Config）
    对象类型注册表
    可视化样式预设
    分析问卷配置（关键帧问答题目）
    数据解析器管理（支持的格式列表）
```

### 2.3 数据库模块详细功能

```
数据库页面
├── 视图切换：列表 / 卡片 / 表格
│
├── 上传
│   单文件上传（db3/db4/noah_txt/csv/...）
│   批量上传（zip 包）
│   URL 导入（从对象存储直接导入）
│   本地客户端同步（大文件场景）
│
├── 检索与过滤
│   关键词搜索（名称、描述、标签）
│   格式过滤（db3 / db4 / csv / ...）
│   状态过滤（上传中 / 处理中 / 就绪 / 归档）
│   时间范围过滤（上传时间 / 数据采集时间）
│   标签过滤（多标签与/或逻辑）
│   有无 Issue 过滤
│
├── 标签管理
│   自由标签 + 预设标签树
│   批量打标签
│   标签统计（各标签下多少录包）
│
├── 录包详情页
│   元数据（时长、大小、格式、采集时间、上传者）
│   包含的对象类型概览
│   关联的 Issue 列表
│   关联的分析会话历史
│   操作：打开分析 / 生成报告 / 分享 / 归档 / 删除
│
└── 数据集（Dataset）
    将多个相关录包组合成数据集
    场景：同一路段不同时间 / 同一算法版本的所有测试数据
    数据集可作为报告和对比分析的单位
```

### 2.4 统计报告模块

```
报告类型
│
├── 单录包报告
│   时间范围内各对象类型的数量统计
│   置信度分布（均值/方差/最低点）
│   问题关键帧列表及分类汇总
│   自动生成文字摘要
│
├── 多录包对比报告
│   选 N 个录包，选指标
│   折线图/柱状图对比
│   自动标注最优/最差录包
│
├── 版本对比报告（A/B 对比）
│   专门用于验证算法修复效果
│   选修复前录包 + 修复后录包
│   关键指标 before/after 对比表
│   结论：改善 / 退化 / 无影响
│
└── 问题统计报告
    选时间范围（如本月）
    Issue 数量趋势图
    Issue 按分类/模块/优先级分布
    平均发现时间、平均解决时长
    未解决 Issue 老龄化分析（长期未关闭的问题）
```

### 2.5 数据流

```
外部数据 → 上传/导入
    ↓
格式解析（Parser）
    ↓
对象提取（Converter → VizObject）
    ↓
存储（对象存储 + 元数据库）
    ↓
可视化分析工作台
    ↓
问题关键帧 → Issue → 问题流转
    ↓
统计报告（聚合分析结果）
```

## 三、项目管理平台

### 3.1 定位

管理算法开发项目的进度，以 Issue 为核心驱动力，自动生成进度报告。

**核心引擎：Plane（开源自托管）**，在其上增加与数据平台的深度集成。

### 3.2 功能模块

```
项目管理平台
│
├── 📋 项目列表
│   新建项目（关联算法模块：定位/感知/规划/控制）
│   项目概览（进度、Issue 数、成员、活跃度）
│
├── 🗂 Issue 看板（来自 Plane）
│   状态：待处理 / 进行中 / 待验证 / 已关闭
│   支持 Kanban / List / Calendar 视图
│   Issue 带数据锚点（可直接跳回录包现场）
│
├── 🏁 里程碑管理
│   里程碑 = 一组 Issue 的集合
│   里程碑进度 = 关联 Issue 关闭比例（自动计算）
│   里程碑关联算法版本（git tag）
│
├── 📈 进度看板（自动生成）
│   本周新增 Issue / 关闭 Issue / 遗留 Issue
│   燃尽图（Issue 数量趋势）
│   各模块 Issue 分布（定位/感知/规划...）
│   里程碑完成进度甘特图
│
└── 🔗 集成管理
    关联代码仓库（GitHub / GitLab）
    PR 自动关联 Issue
    算法版本与录包数据自动绑定
```

### 3.3 Issue 数据模型（扩展标准 Issue）

```typescript
type Issue = {
  // 标准 Issue 字段（Plane 提供）
  id:          string
  title:       string
  description: string
  status:      "open" | "in_progress" | "pending_verify" | "closed"
  priority:    "P0" | "P1" | "P2" | "P3"
  assignee:    UserId
  created_by:  UserId
  created_at:  Date
  closed_at?:  Date

  // 扩展：数据锚点（平台自动填入）
  recording_id:        string         // 来自哪个录包
  time_range:          [number, number] // 哪个时间段
  object_ids:          string[]       // 关联哪些对象
  snapshot_url:        string         // 问题现场截图

  // 扩展：算法上下文（平台自动填入）
  algorithm_module:    string         // 定位 / 感知 / 规划 / 控制
  algorithm_version:   string         // 发现问题时的算法版本（git commit）
  fix_commit?:         string         // 修复的 commit

  // 扩展：验证（与数据平台联动）
  verify_recording_id?: string        // 验证用的录包
  verify_status?:       "pass" | "fail" | "pending"
  verify_report_url?:   string        // 对比报告链接

  // 扩展：来源追踪
  source_keyframe_id:  string         // 来自哪个分析关键帧
  source_session_id:   string         // 来自哪次分析会话
}
```

### 3.5 Issue 生命周期

```
分析人员在数据工作台发现问题
    ↓ 创建问题关键帧 → 一键转为 Issue
Issue 自动携带：录包 / 时刻 / 对象 / 截图 / 算法版本
    ↓ 项目负责人分配给开发者
开发者查看 Issue → 点"查看现场" → 直接跳回录包那一帧
    ↓ 开发者修复代码，提交 PR，关联 Issue ID
PR 合并 → Issue 自动进入"待验证"状态
    ↓ 分析人员在新版本数据上运行验证
数据平台自动生成 before/after 对比报告
    ↓ 验证通过 → Issue 关闭
项目进度自动更新（里程碑完成度 +1）
    ↓
人员管理平台自动记录：谁发现 / 谁修复 / 用了多久
```

## 四、人员管理平台

### 4.1 定位

自动汇总每个人的工作内容，生成工作报告，不需要手动填写日报周报。

**数据完全来自其他两个平台的行为记录**，无额外录入。

### 4.2 数据来源

```
数据来源                    记录什么

数据管理平台
  分析会话           →  谁分析了哪个录包，分析了多长时间
  问题关键帧创建     →  谁发现了问题，发现时间，问题类型
  报告生成           →  谁生成了哪类报告

项目管理平台
  Issue 创建         →  谁提出了问题
  Issue 关闭         →  谁解决了问题，用了多少天
  Issue 评论         →  谁参与了讨论
  PR 关联            →  谁提交了修复代码（从 git 同步）
```

### 4.3 功能模块

```
人员管理平台
│
├── 👥 成员列表
│   成员信息、角色（分析师/算法工程师/管理员）
│   本周工作摘要（自动生成）
│   活跃度指标
│
├── 📋 个人工作台（个人视角）
│   我的分析会话历史
│   我创建的 Issue
│   我负责的 Issue（待处理/进行中）
│   我本周的工作摘要
│
├── 📊 团队效能看板（管理视角）
│   团队整体活跃度
│   Issue 发现/解决人员排行
│   各成员工作量分布
│   工作负载均衡分析
│
└── 📄 工作报告（自动生成）
    个人周报 / 月报
    团队周报 / 月报
    可导出 PDF 或分享链接
```

## 六、技术架构

### 6.1 前端（统一 SPA）

```
统一前端应用（Vue 3 + TypeScript）
│
├── 路由
│   /data/                数据管理平台
│   /data/library         数据库
│   /data/analysis/:id    可视化分析工作台
│   /data/reports         统计报告
│   /data/config          数据配置
│   /project/             项目管理平台
│   /project/:id          项目详情
│   /project/:id/issues   Issue 看板
│   /personnel/           人员管理平台
│   /personnel/me         个人工作台
│   /personnel/team       团队效能
│
└── 共享模块
    认证 / 权限
    通知系统
    全局搜索（跨平台搜索 录包/Issue/成员）
    主题 / 布局
```

### 6.2 后端服务

```
API 网关
│
├── 数据服务（Data Service）
│   录包管理 CRUD
│   对象数据解析与转换
│   VizObject chunk 接口
│   统计报告生成
│
├── 分析服务（Analysis Service）
│   分析会话管理
│   问题关键帧 CRUD
│   分析事件记录
│
├── Issue 服务（集成 Plane API）
│   Issue CRUD
│   与数据锚点的绑定
│   状态流转
│
├── 项目服务（Project Service）
│   项目 / 里程碑管理
│   进度自动计算
│   GitHub / GitLab 集成
│
└── 人员服务（Personnel Service）
    工作数据聚合
    报告自动生成
    权限管理
```

### 6.3 存储

```
对象存储（MinIO / S3 / OSS）
  /recordings/            原始录包文件
  /media/                 图像 / 点云 / 视频帧
  /reports/               生成的报告文件
  /snapshots/             问题关键帧截图

关系数据库（PostgreSQL）
  recordings              录包元数据
  viz_objects             VizObject 注册表
  analysis_sessions       分析会话
  issue_keyframes         问题关键帧
  users / roles           用户与权限

Issue 数据库（Plane 自带）
  projects / issues / milestones / comments
```

## 八、各平台核心指标

| 平台 | 核心价值 | 关键指标 |
|------|----------|----------|
| 数据管理 | 数据资产可管理、可分析 | 录包数量 / 分析覆盖率 / 报告生成数 |
| 项目管理 | 问题发现到解决的闭环速度 | Issue 平均解决时长 / 里程碑达成率 |
| 人员管理 | 工作透明化，减少汇报成本 | 自动报告覆盖率 / 手动填报时间为零 |
